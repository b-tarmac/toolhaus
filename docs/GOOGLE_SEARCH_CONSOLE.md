# Google Search Console Setup

This guide covers submitting Toolhaus to Google Search Console for indexing and search visibility.

## Prerequisites

- Site deployed and live at `https://toolhaus.dev` (or your production domain)
- Sitemap available at `https://toolhaus.dev/sitemap.xml`
- `robots.txt` references the sitemap (already configured in `app/robots.ts`)

## Step 1: Add Your Property

1. Go to [Google Search Console](https://search.google.com/search-console)
2. Click **Add property**
3. Choose **URL prefix** and enter `https://toolhaus.dev`
4. Complete ownership verification using one of:
   - **HTML file upload**: Download the verification file, add to `apps/web/public/`, redeploy
   - **HTML tag**: Add meta tag to `apps/web/app/layout.tsx` (temporary, remove after verification)
   - **DNS record**: Add TXT record to your domain (recommended for production)
   - **Google Analytics**: If already using GA4 with the same property

## Step 2: Submit the Sitemap

1. In Search Console, go to **Sitemaps** (left sidebar)
2. Enter `sitemap.xml` in the "Add a new sitemap" field
3. Click **Submit**

The sitemap URL will be: `https://toolhaus.dev/sitemap.xml`

## Step 3: Verify Sitemap Contents

The sitemap includes:

- Homepage (`/`)
- Tools index (`/tools`)
- Category pages (`/tools/category/[category]`)
- All 25 tool pages (`/tools/[slug]`)

Each URL has `lastModified`, `changeFrequency`, and `priority` set for optimal crawling.

## Step 4: Request Indexing (Optional)

For faster indexing of new or updated pages:

1. Go to **URL Inspection** in Search Console
2. Enter a URL (e.g. `https://toolhaus.dev/tools/json-formatter`)
3. Click **Request indexing**

Use sparingly—Google will crawl regularly once the sitemap is submitted.

## Step 5: Monitor Performance

After submission:

- **Coverage**: Check for crawl errors or excluded pages
- **Performance**: Monitor impressions, clicks, and average position
- **Enhancements**: Review Core Web Vitals and mobile usability

## Sitemap Generation

The sitemap is auto-generated by Next.js from `apps/web/app/sitemap.ts`. It pulls URLs from the tools registry and categories. No manual updates needed when adding new tools—the registry is the source of truth.

## robots.txt

Current `robots.txt` (from `app/robots.ts`):

```
User-agent: *
Allow: /
Disallow: /api/
Disallow: /dashboard/

Sitemap: https://toolhaus.dev/sitemap.xml
```

This allows all crawlers except `/api/` and `/dashboard/`, and points to the sitemap.
